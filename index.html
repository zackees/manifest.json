manifest.json

A codified, proto-defined JSON format for describing downloadable artifacts — asset name, platform tuple, optional sha, download URL, and links to other manifests. One schema serves humans reading the JSON, tools consuming it, and binaries that embed a compiled per-target slice.

Versioned schemas

VersionJSON SchemaProtobuf source
v1 v1/manifest.schema.json v1/manifest.proto

Pin your $schema to the versioned URL above — not the repo HEAD — so your documents stay valid forever as the spec evolves.

Use it in a manifest

{
  "$schema": "https://zackees.github.io/manifest.json/v1/manifest.schema.json",
  "kind": "Catalog",
  "schema_version": 1,
  "tool": "clang",
  ...
}

VS Code (and other JSON-aware editors) auto-validate on save when $schema is set.

Validate locally

pip install git+https://github.com/zackees/manifest.json
manifest-validate path/to/manifest.json
